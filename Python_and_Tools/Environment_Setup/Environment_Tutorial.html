Instalação do Python e configuração de Ambiente Virtual¶

Nessa primeira divisão faremos a instalação do python e a configuração de ambientes virtuais. O python possui um framework open souce (conjunto de ferramentas) disponível chamado Anaconda, ao longo do tutorial faremos a configuração do python e suas bibliotecas de forma nativa e também via anaconda.

Sobre o Anaconda¶

O Anaconda compõe um conjunto de ferramentas para desenvolvimento em Data Science, algumas das suas aplicações incluem uma IDE para python (Spyder), um notebook para python (Jupyter), diversas bibliotecas de machine learning já instaladas (matplotlib, h20, tensorflow, numpy, scipy, etc ..), criação de ambientes virtuais, além disso também há opção de instalação no RStudio.

Verificação de instalação do python¶

Verifique se o python está instalado na sua máquina. Abra o Terminal e digite o código:

In [2]:
python --version

Python 3.7.0

Caso o comando não funcione o python pode não estar instalado ou não foi adicionado nas variáveis de ambiente do seu sistema operacional.

Instalando o python:¶

Se o python não estiver instalado faça download na página oficial de acordo com seu SO ou instale o Anaconda, lembre-se de adicionar o python nas variáveis de ambiente (PATH) do sistema:

Adicionando o python no PATH do sistema operacional (caso não esteja):¶

Windows:¶

Abra o Prompt de Comando e digite: type path %path%;C:\Python

Note que "C:\Python" deve ser substituído por sua pasta de instalação do python.

Linux ou MAC:¶

Abra o Prompt de Comando e digite: type export PATH="$PATH:/usr/local/bin/python"

Um ambiente de desenvolvimento encapula os pacotes que estão sendo usados no desenvolvimento, isso permite que criemos diversos projetos com pacotes e versões diferentes do python e reaproveitemos, por exemplo, em outras projetos ou não tenhamos retrabalhos por incompatibilidade de versões.

No python 3, por default, os pacotes pip e virtualenv (usado para criação do ambiente de desenvolvimento) já vem instalados.

Instalando o pacotes e configurando o ambiente virtual (Se utilizar o Anaconda pode pular essa sessão)¶

O PIP é um gerenciador de pacote, com ele conseguimos instalar, atualizar, remover pacotes em python.

Idealmente coloca-se o pip no path (em SO'S UNIX ele já está lá por default), caso não esteja o processo é o mesmo mencionado acima, a diferença é que sua instalação está em pastaDoPython\Scripts\pip.exe

Se o pip não estiver instalado¶

Acesse: https://bootstrap.pypa.io/get-pip.py

Salve todo o conteudo da página em um notepad com o nome "get-pip.py".

Abra o terminal e execute: python get-pip.py. PIP será instalado.

Se o virtualenv não estiver instalado¶

Para fazer a instalação, no Terminal execute:

pip install virtualenv

(essa é a sintaxe para instalação de bibliotecas com o pip)

Tal como o pip o virtualenv pode ser adicionado ao PATH.

pastaDoPython\Scripts\virtualenv.exe

Criação de um ambiente de desenvolvimento¶

Para criar um ambiente de desenvolvimento com o venv instalado manualmente utilizamos a seguinte sintaxe:

virtualenv ambiente_1

Para criar um ambiente de desenvolvimento com o virtualenv já instalado por default:

python3 -m venv ambiente_1

Criamos um ambiente de desenvolvimento com o nome "ambiente_1" que é uma cópia da versão e pacotes instalados no atual sistema. Para demais opções de configuração do virtualenv (como versão do python a ser isolada, criação do ambiente sem o pip, etc) consultar a documentação em: https://virtualenv.pypa.io/en/latest/

Instalando bibliotecas e configurando o ambiente virtual no Anaconda¶

Idealmente coloca-se o Anaconda no PATH e defini-se isso já na sua instalação, caso isso não ocorra é só fazer esse procedimento manualmente.

O Anaconda tem um gerenciador de pacotes já embutido nele, a sintaxe é bem parecida com o pip. Para instalar um biblioteca:

conda install (nome da bilioteca) Ex: conda install jupyter

Criação de um ambiente de desenvolvimento¶

Para criar um ambiente de desenvolvimento com oconda utilizamos a seguinte sintaxe:

conda create -n ambiente_1

Criamos um ambiente de desenvolvimento com o nome "ambiente_1" que é uma cópia da versão e pacotes instalados no atual sistema. Para demais opções de configuração do ambiente de desenvolvimento do Anaconda (como versão do python , etc) consultar a documentação em: https://conda.io/docs/commands/conda-create.html,

Ativação de um ambiente de desenvolvimento¶

Na linha de comando

source activate ambiente_1 (Linux, MacOSX) activate ambiente_1 (Windows)

Desativação de um ambiente de desenvolvimento¶

source deactivate (Linux, MacOSX) deactivate (Windows)

Mais funções, parâmetros podem ser facilmente consultados no Conda Cheet: https://conda.io/docs/_downloads/conda-cheatsheet.pdf

Environment Setup das aulas (jupyter-notebook)¶

Nossas aulas serão desenvolvidas no jupyter-notebook.

Se você está utilizando o Anaconda ele provavelmente já instalou o jupyter para você, caso não o tenha feito o comando é:

conda install jupyter¶

Se você está utilizando o python nativo a intação deverá ser feita via pip:

pip install jupyter¶

Para iniciar o notebook inserir na linha de comando (ou na visualização gráfica do anaconda):

jupyter-notebook¶

Um notebook será aberto em uma guia do seu navegador padrão.

Para criar um novo documento Clicar em New e então você poderá começar o desenvolvimento do seu Notebook clicando na versão do Python.

Orientações básicas sobre o jupyter¶

  • As células podem contem códigos, texto, imagens, etc. É preciso definir para qual fim será usado cada célula (Markdown, Code).

  • Para executar o conteúdo da célula utiliza-se a botão Run ou CTRL+ENTER. Cada célula é executada individualmente.

  • Todas as variáveis, pacotes e funções, a menos se defina o contrário, são globais. Ná prática isso quer dizer que uma variável criada na primeira célula estará disponível a todas as outras.

  • O Jupyter em si não executa diretamente na linha de comando, um processo em segundo plano realiza no SO os processamentos.

  • De foma periódica o jupyter executa salvamentos automáticos. (2 em 2 minutos).

  • O notebook fica armazenado em um arquivo com extensão ipynb.

  • Objetos colocados na última linha de cada célula (no modo Code) que não possuem alguma atribuição são exibidos como output. Isso é válido para variaveis, plots, dataframes, etc.

  • As bibliotecas instaladas no python do seu ambiente virtual estão disponpivel para utilização no notebook e sua importação é feita normalment via import.

  • Caso alguma bibliotecas necessite ser instalada faz-se isso ou na linha de comando do python, ou no jupyter utilizando um "!" na frente, isso faz com que esse comando seja executado em Shell. Ex: !pip install pandas.
