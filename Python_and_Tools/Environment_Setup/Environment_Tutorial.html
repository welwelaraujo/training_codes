Instalação do Python e configuração de Ambiente Virtual¶

Nessa primeira diviso faremos a instalao do python e a configurao de ambientes virtuais. O python possui um framework open souce (conjunto de ferramentas) disponvel chamado Anaconda, ao longo do tutorial faremos a configurao do python e suas bibliotecas de forma nativa e tambm via anaconda.

Sobre o Anaconda¶

O Anaconda compe um conjunto de ferramentas para desenvolvimento em Data Science, algumas das suas aplicaes incluem uma IDE para python (Spyder), um notebook para python (Jupyter), diversas bibliotecas de machine learning j instaladas (matplotlib, h20, tensorflow, numpy, scipy, etc ..), criao de ambientes virtuais, alm disso tambm h opo de instalao no RStudio.

Verificação de instalação do python¶

Verifique se o python est instalado na sua mquina. Abra o Terminal e digite o cdigo:

In [2]:
python --version

Python 3.7.0

Caso o comando no funcione o python pode no estar instalado ou no foi adicionado nas variveis de ambiente do seu sistema operacional.

Adicionando o python no PATH do sistema operacional (caso não esteja):¶

Windows:¶

Abra o Prompt de Comando e digite: type path %path%;C:\Python

Note que "C:\Python" deve ser substitudo por sua pasta de instalao do python.

Linux ou MAC:¶

Abra o Prompt de Comando e digite: type export PATH="$PATH:/usr/local/bin/python"

Um ambiente de desenvolvimento encapula os pacotes que esto sendo usados no desenvolvimento, isso permite que criemos diversos projetos com pacotes e verses diferentes do python e reaproveitemos, por exemplo, em outras projetos ou no tenhamos retrabalhos por incompatibilidade de verses.

No python 3, por default, os pacotes pip e virtualenv (usado para criao do ambiente de desenvolvimento) j vem instalados.

Instalando o pacotes e configurando o ambiente virtual (Se utilizar o Anaconda pode pular essa sessão)¶

O PIP um gerenciador de pacote, com ele conseguimos instalar, atualizar, remover pacotes em python.

Idealmente coloca-se o pip no path (em SO'S UNIX ele j est l por default), caso no esteja o processo o mesmo mencionado acima, a diferena que sua instalao est em pastaDoPython\Scripts\pip.exe

Se o pip não estiver instalado¶

Acesse: https://bootstrap.pypa.io/get-pip.py

Salve todo o conteudo da pgina em um notepad com o nome "get-pip.py".

Abra o terminal e execute: python get-pip.py. PIP ser instalado.

Se o virtualenv não estiver instalado¶

Para fazer a instalao, no Terminal execute:

pip install virtualenv

(essa a sintaxe para instalao de bibliotecas com o pip)

Tal como o pip o virtualenv pode ser adicionado ao PATH.

pastaDoPython\Scripts\virtualenv.exe

Criação de um ambiente de desenvolvimento¶

Para criar um ambiente de desenvolvimento com o venv instalado manualmente utilizamos a seguinte sintaxe:

virtualenv ambiente_1

Para criar um ambiente de desenvolvimento com o virtualenv j instalado por default:

python3 -m venv ambiente_1

Criamos um ambiente de desenvolvimento com o nome "ambiente_1" que uma cpia da verso e pacotes instalados no atual sistema. Para demais opes de configurao do virtualenv (como verso do python a ser isolada, criao do ambiente sem o pip, etc) consultar a documentao em: https://virtualenv.pypa.io/en/latest/

Instalando bibliotecas e configurando o ambiente virtual no Anaconda¶

Idealmente coloca-se o Anaconda no PATH e defini-se isso j na sua instalao, caso isso no ocorra s fazer esse procedimento manualmente.

O Anaconda tem um gerenciador de pacotes j embutido nele, a sintaxe bem parecida com o pip. Para instalar um biblioteca:

conda install (nome da bilioteca) Ex: conda install jupyter

Criação de um ambiente de desenvolvimento¶

Para criar um ambiente de desenvolvimento com oconda utilizamos a seguinte sintaxe:

conda create -n ambiente_1

Criamos um ambiente de desenvolvimento com o nome "ambiente_1" que uma cpia da verso e pacotes instalados no atual sistema. Para demais opes de configurao do ambiente de desenvolvimento do Anaconda (como verso do python , etc) consultar a documentao em: https://conda.io/docs/commands/conda-create.html,

Ativação de um ambiente de desenvolvimento¶

Na linha de comando

source activate ambiente_1 (Linux, MacOSX) activate ambiente_1 (Windows)

Desativação de um ambiente de desenvolvimento¶

source deactivate (Linux, MacOSX) deactivate (Windows)

Mais funes, parmetros podem ser facilmente consultados no Conda Cheet: https://conda.io/docs/_downloads/conda-cheatsheet.pdf

Orientações básicas sobre o jupyter¶

  • As clulas podem contem cdigos, texto, imagens, etc. preciso definir para qual fim ser usado cada clula (Markdown, Code).

  • Para executar o contedo da clula utiliza-se a boto Run ou CTRL+ENTER. Cada clula executada individualmente.

  • Todas as variveis, pacotes e funes, a menos se defina o contrrio, so globais. N prtica isso quer dizer que uma varivel criada na primeira clula estar disponvel a todas as outras.

  • O Jupyter em si no executa diretamente na linha de comando, um processo em segundo plano realiza no SO os processamentos.

  • De foma peridica o jupyter executa salvamentos automticos. (2 em 2 minutos).

  • O notebook fica armazenado em um arquivo com extenso ipynb.

  • Objetos colocados na ltima linha de cada clula (no modo Code) que no possuem alguma atribuio so exibidos como output. Isso vlido para variaveis, plots, dataframes, etc.

  • As bibliotecas instaladas no python do seu ambiente virtual esto disponpivel para utilizao no notebook e sua importao feita normalment via import.

  • Caso alguma bibliotecas necessite ser instalada faz-se isso ou na linha de comando do python, ou no jupyter utilizando um "!" na frente, isso faz com que esse comando seja executado em Shell. Ex: !pip install pandas.
